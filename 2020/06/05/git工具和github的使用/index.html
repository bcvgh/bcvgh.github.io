.git命令行使用

git分布式版本控制软件是通过

工作区—本地服务器—远程仓库的提交方式

git仓库有ssh和https两种方式克隆到本地，其中通过https方式的话传输过程中经常需要验证身份，操作不方便，所以 文章采用ssh配置公钥私钥的方式。

配置公钥私钥

git config --list查看所有配置项

创建私钥sssh-keygen -t rsa -C(磁盘名)

找到生成私钥的文件价，发现产生了两个私钥信息文件

查看内容并复制到github-setting-SSH and GPC key 中去

cat ~/.ssh/id_rsa.pub查看文件内容

点击Add SSH key ，公钥配置成功。

接下来实现远程建立git项目，从github仓库下克隆一个项目到本地

git clone xxx.git 克隆仓库和本地同步。

由于我这个项目是一个新项目，所以只有一个readme.md文件

切换到到本地仓库的目录并执行git status命令查看仓库状态。（当前状态表示仓库没有在进行任何操作）

当你在本地仓库中新建一个文件,再执行git status发现状态现在增加了一个新的文件，需要git add<file>命令提交

)

输入git add ceshi.html或者git add .（表示提交所以改动的文件）将文件提交到暂存区

在查看状态时新增的文件从红色变为绿色，表示文件已经再暂存区，需要commit命令提交到本地服务区git commit -m "需要添加的注释"。（如果文件只能内容上的更改，并没有建立新文件，可以使用commit -a -m "需要添加的注释“直接从工作区提交到本地服务器）

最后一步，输入git push origin master提交到远程服务器(origin:远程服务器名称，master：远程仓库上的主分支)

再到github上查看仓库时发现仓库内容已经经过的更新，其中commits变为两次(commits:项目提交次数)。

git还有一个使用的功能，叫做git分支，它可以从主线上分离出来做另外的操作而又不影响主线(多用于解决项目的临时序求)

git branch 分支名称新建分支

git checkout 分支名称切换分支(只有切换分支以后，对仓库内容的改动才会在新分支上进行，不然一直是在主分支master上进行)

接下来在新分支上新建了一个新文件,并向往常命令一样将文件上传到暂存区—提交到本地服务器—上传到远程仓库

接下来查看github上项目仓库，发现新分支上的文件比主分支上多一个ch1_cehshi.html文件

项目没有问题，只要将分支合并一个项目就完成了。
